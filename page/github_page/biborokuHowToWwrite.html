📝 備忘録の書き方

このページでは、備忘録サイトの構成や編集方法をまとめています。

📌 使用しているツール

  • 💎 Ruby - Jekyllの動作に必要
  • 📄 Jekyll - 静的サイトジェネレータ
  • 🛠 Git - バージョン管理ツール
  • 🌍 GitHub Pages - サイト公開サービス

🚀 ローカルで備忘録を確認したい場合

  1. 💻 ローカルサーバーを起動： 
    bundle exec jekyll serve
  2. 🌐 ローカルでサイトを表示： 
    http://127.0.0.1:4000/

🔄 GitHub Pagesに反映されない場合の対処法

ローカルでは変更が反映されるのに、GitHub Pages上では更新されない場合の解決策。

🧹 キャッシュの削除

ブラウザキャッシュが原因で古いページが表示される場合があります。以下を試してください：

  • 🔄 Ctrl + Shift + R で強制リロード
  • 🕵️ シークレットモードでアクセス
  • 📱 別のデバイス（スマホなど）で確認

🚀 GitHub Pagesのデプロイを強制リロード

GitHubのデプロイ処理が遅延している場合は、以下のコマンドで強制的に再デプロイします。 

git commit --allow-empty -m "Trigger GitHub Pages redeploy"
git push origin main

🔧 Jekyllをリビルドする方法

Jekyllの変更がローカル環境やGitHub Pagesに反映されない場合、リビルドを行います。

🗑️ キャッシュの削除

  • Jekyllのビルドキャッシュが原因で変更が適用されない場合があります。
  • 以下のコマンドを実行してキャッシュを削除してください：
bundle exec jekyll clean

⚙️ Jekyllを再起動

  • キャッシュを削除した後、Jekyllを再起動して変更を反映させます。
  • 以下のコマンドを実行してください：
bundle exec jekyll serve
