💻 VSCodeについて

このページでは、VSCodeの便利な機能や拡張機能についてまとめています。

🔧 VSCodeの便利な機能

  • 🌸 Japanese Language Pack for Visual Studio Code:

    Visual Studio Codeを日本語化するための拡張機能です。メニューやUIが日本語表示になるので、より使いやすくなります。

    1. 拡張機能のインストール:「Extensions」タブで「Japanese Language Pack」を検索し、インストールします。
    2. 表示言語の設定:VSCodeで「Ctrl + Shift + P」を押してコマンドパレットを開き、「Configure Display Language」を選択します。日本語を選ぶことで言語が切り替わります。
    Visual Studio Marketplaceで詳細を見る
  • ✨ Prettier:

    コードを自動的に整形してくれる拡張機能で、一貫性を保ちながら効率よく作業できます。保存時に自動でコードを美しく整形する機能があり、チームでの共同作業にも役立ちます。

    1. 拡張機能のインストール:「Extensions」タブで「Prettier」を検索し、「Prettier – Code formatter」をインストールします。
    2. 保存時の自動整形を有効化:VSCode設定でeditor.formatOnSaveをtrueに設定し、保存時にコードが自動整形されるようにします。
    3. デフォルトフォーマッターを設定:editor.defaultFormatterでesbenp.prettier-vscodeを選択し、Prettierを使用するよう指定します。
    4. カスタム設定:.prettierrcファイルで独自の整形ルールを設定し、プロジェクトに合わせて調整します。
    5. 補足: 一部だけフォーマッターをかけたい場合は、VSCodeで対象コードを選択し、右クリックメニューから「フォーマッターでフォーマット」を選択します。または、選択範囲だけフォーマットするショートカット（WindowsならShift + Alt + F）を使用することも可能です。 ※これらの方法は、VSCodeが提供する基本的な機能であり、Prettier独自の機能ではありません。
    Visual Studio Marketplaceで詳細を見る
  • 💡 Workspace Explorer:

    VSCode上でワークスペースの切り替えを簡単に管理できる拡張機能です。複数のプロジェクトを効率的に切り替えたり、ワークスペースを保存して整理することができます。

    1. 拡張機能のインストール:「Extensions」タブで「Workspace Explorer」を検索し、インストールします。
    2. ワークスペース保存ディレクトリの設定:VSCode設定でworkspaceExplorer.workspaceStorageDirectoryを設定し、保存場所を指定します。
    3. ワークスペースを作成:「File」→「名前を付けてワークスペースを保存」でプロジェクトを保存します。
    4. 拡張機能の管理:左ペインの「WORKSPACES」タブでワークスペースを切り替えたり、新しいウィンドウで開いたりできます。
    Visual Studio Marketplaceで詳細を見る
  • 🛠️ Angular Language Service:

    Angular専用の拡張機能で、テンプレートのオートコンプリート、エラーの即時検出、定義へのジャンプなどを可能にします。コード品質の向上にも役立つ便利なプラグインです。

    1. strictTemplatesオプション:

      tsconfig.jsonに以下を追加すると、テンプレート型チェックを強化してバグを防ぐ効果があります： 

          {
      "angularCompilerOptions": {
        "strictTemplates": true
      }
    }
    2. 設定例:

      VSCodeのsettings.jsonでは、以下の設定例でAngular Language Serviceをさらにカスタマイズできます： 

          {
      "editor.formatOnSave": true,
      "angular.languageService.experimental": true
    }
    Visual Studio Marketplaceで詳細を見る
  • 🛠️ Gradle Language Service:

    Gradle用の拡張機能で、ビルドスクリプトの作業効率をぐっと高める便利なツールです。以下が主な特徴です：

    • 📜 コード補完: Gradleのビルドスクリプト内で、賢く候補を提示。
    • ⚠️ エラー表示: スクリプト内のミスを即座にハイライト。
    • 🔗 ジャンプ機能: 依存関係やタスク定義へのクイックジャンプが可能。
    • 🚀 タスク実行: 拡張機能から直接Gradleタスクを実行できる。

    プロジェクトディレクトリにbuild.gradleまたはbuild.gradle.ktsファイルがあると、自動的に機能が有効化されます。VSCodeの設定ファイルで以下の調整もできます： 

          {
        "gradle.java.home": "/path/to/java/home",
        "gradle.enableTaskQuickPick": true
      }

  • 🛠️ Extension Pack for Java:

    Java開発環境を効率化するための拡張機能パックです。以下が主な特徴です：

    • 📜 Language Support for Java by Red Hat: コード補完やリファクタリングを提供。
    • 🐞 Debugger for Java: Javaアプリケーションのデバッグを簡単に。
    • 🧪 Java Test Runner: JUnitやTestNGによるテスト実行とデバッグをサポート。
    • ⚙️ Maven for Java: Mavenプロジェクトの管理を効率化。
    • 🔗 Gradle for Java: Gradleタスクの管理と実行を簡易化。
    • 🗂️ Project Manager for Java: プロジェクト整理やクラス管理を便利に。
    • ✨ Visual Studio IntelliCode: AIによるコード補完を提供し、生産性向上を支援。

    ⚙️ 初期設定が必要な場合

    Extension Pack for Javaを正しく動作させるには、以下の手順を行う必要があります：

    1. JDK（Java Development Kit）のインストール: 必要なJDKをインストールし、`java -version`または`javac -version`で確認してください。
    2. Java Homeの設定: 環境変数`JAVA_HOME`にJDKのルートディレクトリを設定します。
      • 例: Windowsの場合、`C:\Program Files\Java\jdk-XX.X.X`
      • **Path変数に追加:** `%JAVA_HOME%\bin`
      • Mac/Linuxでは以下を設定: 
              export JAVA_HOME=/path/to/your/jdk
      export PATH=$JAVA_HOME/bin:$PATH
    3. プロジェクトの初期セットアップ: VSCodeのコマンドパレットで「Java: Create Java Project」を選択し、プロジェクトを作成します。

  • 🛠️ Lombok Annotation Support:

    Javaの開発で便利なLombokライブラリをVSCodeでも簡単に扱えるようにする拡張機能です。以下が主な特徴です：

    • 📜 コード補完とリファクタリング: Lombokのアノテーション（例: @Getter, @Setter）を利用したコード補完をサポート。
    • 🔄 Delombok機能: Lombokのアノテーションを通常のJavaコードに変換。標準的なgetter/setterやコンストラクタを自動生成したコードとして出力可能。
    • 🛠️ 対応する注釈: 主な対応するLombokアノテーションは以下の通り：
      • @Getter / @Setter
      • @ToString
      • @EqualsAndHashCode
      • @AllArgsConstructor / @NoArgsConstructor
      • @Data

    ⚙️ 初期設定が必要な場合

    以下の手順でプロジェクトにLombokを導入します：

    1. 依存関係を追加:

      Gradleプロジェクトの場合、build.gradleに以下を追加します： 

            dependencies {
          compileOnly 'org.projectlombok:lombok:1.18.38'
          annotationProcessor 'org.projectlombok:lombok:1.18.38'
      }
      Mavenの場合は、pom.xmlに以下の依存関係を記載します： 
            
          org.projectlombok
          lombok
          1.18.38
          provided
    2. Java Homeの設定: 環境変数JAVA_HOMEにJDKのルートディレクトリを指定します。
      • 例: Windowsの場合はC:\Program Files\Java\jdk-XX.X.X
      • **Path変数に追加:** %JAVA_HOME%\bin
      • Mac/Linuxでは以下を設定: 
              export JAVA_HOME=/path/to/your/jdk
      export PATH=$JAVA_HOME/bin:$PATH
    3. Delombokを利用: コマンドラインで以下を実行することで、Lombokアノテーションを削除し、通常のJavaコードを生成します： 
            java -jar lombok.jar delombok src -d src-delomboked

    📝 LombokとDelombokのサンプルコード

    以下は、Lombokを使用した場合とDelombokを適用した場合のコードとコンパイル結果です：

    🔧 Lombokを使用した場合

    ソースコード:

          import lombok.Getter;
      import lombok.Setter;
      
      public class MyClass {
          @Getter @Setter
          private String name;
      
          @Getter @Setter
          private int age;
      }

    コンパイル後の結果:

          public class MyClass {
          private String name;
          private int age;
      
          public String getName() {
              return this.name;
          }
      
          public void setName(String name) {
              this.name = name;
          }
      
          public int getAge() {
              return this.age;
          }
      
          public void setAge(int age) {
              this.age = age;
          }
      }

    🔧 Delombokを使用した場合

    ソースコード（Delombokで変換後）:

          public class MyClass {
          private String name;
          private int age;
      
          public String getName() {
              return this.name;
          }
      
          public void setName(String name) {
              this.name = name;
          }
      
          public int getAge() {
              return this.age;
          }
      
          public void setAge(int age) {
              this.age = age;
          }
      }

    コンパイル後の結果:

          public class MyClass {
          private String name;
          private int age;
      
          public String getName() {
              return this.name;
          }
      
          public void setName(String name) {
              this.name = name;
          }
      
          public int getAge() {
              return this.age;
          }
      
          public void setAge(int age) {
              this.age = age;
          }
      }

    🔄 LombokとDelombokの違い

    Lombokは開発中の効率性重視の選択肢であり、コード記述を簡潔に保てます。一方、Delombokは特定の場面（例えば、Javadoc生成や互換性確保）で必要に応じて使用されるものです。

    主なDelombokコマンド:

    • ディレクトリ全体をDelombok: 
            java -jar lombok.jar delombok src -d delombok-src
    • 単一ファイルをDelombok: 
            java -jar lombok.jar delombok -p MyClass.java
    • フォーマットを整えた出力: 
            java -jar lombok.jar delombok -f pretty src -d delombok-src

  • 🛠️ Spring Boot Extension Pack:

    Spring Bootアプリケーションを効率的に開発するためのVSCode用拡張機能パックです。以下が主な特徴です：

    • 🚀 Spring Initializr: プロジェクトのひな型を簡単に生成でき、言語や依存ライブラリを選択してカスタマイズ可能。
    • 🛠️ Spring Boot Tools: アプリケーションの実行、停止、デバッグをサポート。Spring固有のナビゲーションや補完機能を提供。
    • 📊 Spring Boot Dashboard: サイドバーにSpring専用のダッシュボードを表示し、プロジェクト管理を効率化。

    ⚙️ 初期設定が必要な場合

    Spring Boot Extension Packを最大限活用するために以下の準備が必要です：

    1. Java Development Kit (JDK) のインストール: JDKをインストールし、ターミナルでjava -versionを実行して確認します。バージョン17以上がおすすめ。
    2. Spring Initializrでのプロジェクト作成:
      1. コマンドパレットで「Spring Initializr」を検索。
      2. 言語、バージョン、Group ID、Artifact IDなどをウィザードで設定してプロジェクトを生成。
    3. プロパティファイルの設定:

      プロジェクト内のapplication.propertiesやapplication.ymlを編集してアプリケーションの挙動をカスタマイズ。

      • サーバーポートの変更: server.port=8081
      • データベース接続設定: spring.datasource.url=jdbc:mysql://localhost:3306/mydb
      • ログ設定: logging.level.root=WARN
      • メールサーバー設定: spring.mail.host=smtp.example.com

  • 🛠️ GitLens:

    VSCodeでGit操作を効率化するための拡張機能です。GitLensを使用することで、リポジトリの履歴やブランチ操作を直感的に行えるようになります。

    • 🔍 コード変更履歴の確認:

      「Git Blame」機能を使い、コードの誰がどの部分を変更したかを表示。

    • 📜 コミットの管理:

      過去のコミットメッセージや変更箇所を簡単に確認。特定のコミット間の比較も可能。

    • 🔄 ブランチ間の比較:

      ブランチ間のコード変更を視覚的に確認して、マージ前のチェックが可能。
      サイドバーの「BRANCHES」セクションでブランチを右クリックし、Select for Compareを選択して比較できます。

    • 🌐 リモート管理:

      リモートリポジトリとのフェッチやプッシュを簡単に実行。「↑」アイコンをクリックして変更を反映可能。

    • 🌱 ブランチの作成:

      新しいブランチを作成するには、サイドバーの「BRANCHES」ビューで右クリックして、Create Branchを選択。新しいブランチでの開発やテストがスムーズに行えます。

    • ✨ 課題関連付け:

      GitHubやGitLabの課題をブランチに関連付けて管理可能。ただし、Redmineなどのツールとは直接的な連携はありません。

    🔧 使い方の補足

    GitLensを使うと、変更履歴やブランチ操作が視覚化され、開発効率が大幅に向上します。コミットやプッシュ、ブランチ比較をシームレスに実行できるため、複雑なリポジトリ管理も簡単です。
